# 测试文件保存规则

## 适用场景

使用 dnSpy MCP 工具进行功能验证、回归测试等操作时。

## 规则

1. **测试对象选择**：优先使用文件名含 `copy` 的副本文件（如 `Newtonsoft.Json copy.dll`），禁止对原始文件执行写入操作。
2. **原位写入**：测试中涉及的写入操作（重命名、添加成员、IL 编辑等），完成后一律**原位 `save_assembly` 保存回同一路径**，不输出到新文件。`copy` 后缀已表明其为可覆写的副本。
3. **保存时机**：所有修改操作完成后，必须立即调用 `save_assembly` 持久化，不得仅依赖内存状态做结论性验证。
4. **验证方式**：保存后重新 `load_assembly` 同一文件，通过查询工具（如 `list_members`）确认修改已落盘，方可报告测试通过。
